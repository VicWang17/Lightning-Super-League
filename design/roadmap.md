# 闪电超级联赛 - 开发路线图

本文档定义游戏的开发阶段和里程碑规划，按照由核心到扩展、由基础到深度的原则进行迭代开发。

---

## 版本命名规则

- **M** (Milestone)：核心里程碑版本，代表一个可玩的完整阶段
- **v** (Version)：里程碑内的迭代版本
- 格式：`M1-v1.0`、`M1-v1.1` 等

---

## M1 - 核心基础版本

**目标**：搭建游戏核心框架，实现最基础的联赛循环和比赛体验。

**时间预估**：6-8周

### M1-v1.0 基础设施

**前端**
- [ ] 项目基础架构搭建（技术选型、目录结构、路由配置）
- [ ] UI组件库搭建（按钮、卡片、表单、导航等基础组件）
- [ ] 全局样式系统（颜色、字体、间距规范）

**后端**
- [ ] 项目基础架构搭建
- [ ] 数据库设计（用户表、球队表、球员表、比赛表等核心表结构）
- [ ] API接口规范定义

### M1-v1.1 用户系统

**功能开发**
- [ ] 用户注册（邮箱验证）
- [ ] 用户登录/登出
- [ ] 个人中心基础信息展示
- [ ] 球队创建/选择流程

### M1-v1.2 联赛与赛季系统（基础）

**功能开发**
- [ ] 联赛数据结构（4个联赛、32支球队基础数据）
- [ ] 赛季定时系统框架（比赛时间计算、赛季进度追踪）
- [ ] 赛程生成算法（双循环14场赛制）
- [ ] 积分榜计算与展示

### M1-v1.3 比赛系统（基础）

**功能开发**
- [ ] 比赛模拟引擎V1（基于基础能力值的简单模拟）
- [ ] 文字直播系统（基础事件播报）
- [ ] 比赛结果存储

### M1-v1.4 经济系统（基础）

**功能开发**
- [ ] 球队资金系统（基础资金账户）
- [ ] 基础收入/支出记录
- [ ] 赛季分红发放

**M1验收标准**：
- 用户可以注册登录并创建球队
- 系统能自动按时间表进行比赛并生成结果
- 用户可以查看积分榜和赛程
- 赛季可以正常结束并结算

## M2 - 核心玩法版本

**目标**：完善核心游戏体验，让玩家能够深度管理球队和参与转会市场。

**时间预估**：8-10周

### M2-v1.0 球队管理系统

**球员管理**
- [ ] 球员列表展示（筛选、排序）
- [ ] 球员详情页（属性、状态、合同信息）
- [ ] 基础球员操作（续约、解约）

**战术设置**
- [ ] 可视化战术板
- [ ] 阵型选择（4-4-2、4-3-3、3-5-2等）
- [ ] 比赛心态设置（进攻/平衡/防守）
- [ ] 基础战术指令

### M2-v1.1 交易系统

**转会市场**
- [ ] 球员搜索与筛选
- [ ] 球员挂牌出售
- [ ] 报价与谈判流程
- [ ] 转会窗口期控制

### M2-v1.2 比赛系统（增强）

**模拟引擎升级**
- [ ] 战术影响比赛结果
- [ ] 球员状态影响发挥
- [ ] 主客场因素
- [ ] 更多随机事件（受伤、红黄牌等）

**赛后数据**
- [ ] 详细比赛统计
- [ ] 球员评分系统
- [ ] 比赛回放/事件回顾

### M2-v1.3 训练系统

**功能开发**
- [ ] 训练计划制定
- [ ] 球员专项训练
- [ ] 训练效果反馈
- [ ] 体能恢复机制

### M2-v1.4 新闻生成系统

**功能开发**
- [ ] 比赛结果自动生成战报
- [ ] 转会新闻报道
- [ ] 新闻中心页面

**M2验收标准**：
- 玩家可以完整管理球队阵容和战术
- 转会市场功能完整可用
- 比赛有战术深度和详细赛后数据
- 训练系统能有效提升球员能力

**目标**：增加游戏的策略深度和长期可玩性。

**时间预估**：6-8周

### M3-v1.0 职员系统

**功能开发**
- [ ] 职员类型与数据结构
- [ ] 职员雇佣/解雇
- [ ] 职员能力加成实现
- [ ] 职员薪资管理

### M3-v1.1 青训营系统

**功能开发**
- [ ] 球探派遣系统
- [ ] 年轻球员生成算法
- [ ] 青训球员培养
- [ ] 提拔至一线队流程

### M3-v1.2 升降级系统

**功能开发**
- [ ] 赛季结束升降级逻辑
- [ ] 球队联赛迁移
- [ ] 升降级动画/公告

### M3-v1.3 经济系统（完善）

**功能开发**
- [ ] 比赛日收入（门票、转播）
- [ ] 商业收入系统
- [ ] 工资帽机制
- [ ] 财政公平规则
- [ ] 财务报表

**M3验收标准**：
- 职员系统能有效提升球队各方面能力
- 青训营可持续产出年轻球员
- 升降级机制正常运行

## 风险评估与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 比赛引擎平衡性难调 | 高 | 高 | 尽早开始内部测试，建立数据监控 |
| 定时系统复杂度超预期 | 中 | 高 | 采用成熟定时任务方案，充分测试边界情况 |
| 经济系统通货膨胀 | 中 | 高 | 设置硬上限，持续监控经济数据 |
| 开发周期延误 | 高 | 中 | 优先保证M1/M2核心功能，M3/M4可裁剪 |

- 杯赛 (Cup)
