# 战术系统平衡性测试报告

> 测试时间：2026-05-05
> 测试范围：match-engine 内部引擎战术系统
> 样本规模：单参数测试约 12×5×40 = 2400 场；预设战术矩阵 12×11×30 = 3960 场；组合测试约 2000 场

---

## 一、已修复的代码缺陷

### 1. MarkingStrategy 不一致（selector.go ↔ tactical.go）

**问题**：`selector.go:103` 使用 `== 1` 触发人盯人奖励，而 `tactical.go:38` 使用 `>= 2` 激活 `ManMarkingActive`。
- 结果：混合防守(MarkingStrategy=1)获得了选择器奖励（CB/DMF 权重 2.2×），但人盯人(MarkingStrategy=2)反而没有。

**修复**：
- `selector.go`：`>= 2`（纯人盯人才获得选择器奖励）
- `tactical.go`：`>= 1`（混合防守也激活 ManMarkingActive 标记）

**验证**：修复后纯人盯人胜率 42.4%，混合 36.6%，区域 36.6%，符合设计意图。

### 2. TacklingAggression 犯规严重性计算反向

**问题**：`simulator.go:2749` 使用 `float64(4-agg) * 1.5`，导致侵略性越低，犯规越严重。

**修复**：改为 `float64(agg) * 2.0`，侵略性越高，犯规严重性越高。

## 二、是否存在"绝对好/坏"的选项？

### ⚠️ 存在明显弱势选项

| 参数 | 弱势值 | 胜率 | 强势值 | 胜率 | SWING |
|------|--------|------|--------|------|-------|
| **AttackTempo** | **4 (极速反击)** | **27.5%** | 3 | 55.0% | **27.5%** |
| **PlaymakerFocus** | **4** | **20.0%** | 1 | 42.5% | **22.5%** |
| **AttackWidth** | 2 | 35.0% | 0/3 | 50.0% | 15.0% |
| **PressingIntensity** | 1 | 30.0% | 2 | 55.0% | 25.0% |

**分析**：

1. **AttackTempo=4（极速反击）是最大问题**
   - 虽然触发了大量反击事件（Counters 11.0/match，远超正常的 2.6-3.3），但进球数反而最低（1.58/match）。
   - 原因：`ComputeRiskIndex` 中 AttackTempo 权重 0.20，导致风险指数过高，有压力时激进传球概率大增，失误率上升。CounterBoost 提供的控球偏移（+0.1）和动量（+0.12）不足以弥补高失误带来的球权损失。
   - **结论**：极速反击在当前机制下是"陷阱选项"。

2. **PlaymakerFocus=4 是第二大问题**
   - 射门权重被削减 30%（`1 - 4×0.075 = 0.7`），传球权重增加 32%。
   - 虽然传球更多了，但终结能力严重不足。测试显示 PlaymakerFocus=0 时进球 2.16/match，而 PlaymakerFocus=4 时仅 1.72/match。
   - **结论**：高组织核心权重过度惩罚了射门，导致"只传不射"。

3. **PressingIntensity 和 ShootingMentality 呈现"中间最优"**
   - PressingIntensity=2 最佳（55.0%），极端值 0/1/4 都只有 30-35%。
   - ShootingMentality=2 最佳（57.5%），极端值 0/4 只有 42-48%。
   - **结论**：这是健康的设计——极端逼抢容易耗尽体力/暴露空当，极端射门倾向容易浪射。但当前最优区间太窄，两端值几乎不可用。

### ✅ 相对平衡的参数

| 参数 | SWING | 评价 |
|------|-------|------|
| DefensiveLineHeight | 7.5% | 较为平衡，各值差异不大 |
| DefensiveCompactness | 5.0% | 平衡，但影响偏弱 |
| MarkingStrategy | 2.5%-27.5% | 修复后纯人盯人略强，但总体影响可控 |
| OffsideTrap | 17.5% | OffsideTrap=0 反而最好，说明越位陷阱机制收益不足 |

## 三、版本强势/弱势配置

### 预设战术排名（基于 30 场对局）

| 排名 | ID | 战术名称 | 平均胜率 | 评价 |
|------|-----|---------|---------|------|
| 1 | T10 | 密集防守 | 44.5%-49.1% | 🟢 版本强势 |
| 2 | T08 | 控球消耗 | 33.0%-44.2% | ⚠️ 波动大，有时很强有时很弱 |
| 3 | T06 | 深度防反 | 41.2%-43.0% | 🟢 稳健强势 |
| 4 | T04 | 边路传中 | 40.9%-42.1% | 🟡 中等偏上 |
| 5 | T01 | 传控渗透 | 37.9%-38.2% | 🟡 中等 |
| ... | ... | ... | ... | ... |
| 10 | T09 | 快速反击 | 34.8% | 🔴 偏弱 |
| 11 | T11 | 均衡默认 | 34.5% | 🔴 均衡反而偏弱（说明专精战术收益过高）|
| 12 | T12 | 全攻全守 | 30.0%-36.4% | 🔴 版本弱势 |

**关键发现**：
- **T11（均衡默认）胜率偏低**：均衡战术作为基准，理论上应该接近 50%，但实际只有 34-37%。这说明专精某一方向的战术收益过高，"什么都有一点"反而没有竞争力。
- **T10（密集防守） consistently 最强**：高紧凑度(2) + 人盯人(2) + 越位陷阱(2) + 低防线(1) 的组合在当前机制下非常有效。
- **T12（全攻全守） consistently 最弱**：所有参数都偏高但不够极端，没有突出的优势方向。

### 阵型×战术协同效应

| 战术 | 最佳阵型 | 胜率 | 最差阵型 | 胜率 | SWING |
|------|---------|------|---------|------|-------|
| 均衡 | F03 | 50.0% | F07 | 16.7% | **33.3%** |
| 高位逼抢 | F01 | 58.3% | F02 | 25.0% | **33.3%** |
| 深度防反 | F07 | 54.2% | F04 | 29.2% | **25.0%** |
| 全攻 | F08 | 58.3% | F01 | 20.8% | **37.5%** |

**关键发现**：
- 阵型对胜率的影响**极其巨大**，SWING 可达 25-37%。
- 同一战术在不同阵型下可能从"强势"变成"废物"。
- F07（Diamond Control）在均衡战术下胜率仅 16.7%，但在深度防反下却高达 54.2%。
- F08（Dual Wing）在均衡战术下 37.5%，但在全攻战术下 58.3%。

## 四、进一步优化方案

### 方案 A：修复极端选项（高优先级）

#### A1. 增强 AttackTempo=4 的反击收益
```go
// simulator.go applyCounterBoost
func (sim *Simulator) applyCounterBoost(ms *domain.MatchState, side domain.Side) {
    ...
    ms.CounterBoostRemaining[idx] = 3  // 保持 3 个事件
    sim.boostGlobalMomentum(ms, 0.12)  // 当前值
    // 建议增加：成功后立即的区域推进概率更高
    // 建议增加：反击事件权重从 15 提升到 20-25
}
```
**或者**：降低 AttackTempo 对 RiskIndex 的贡献（从 0.20 降至 0.12-0.15），让极速反击不至于过于激进。

#### A2. 削弱 PlaymakerFocus 对射门的惩罚
```go
// simulator.go
shotReduce := 1.0 - float64(pf)*0.05  // 从 0.075 降至 0.05
// 这样 pf=4 时射门权重为 0.8（削减 20%），而非当前的 0.7（削减 30%）
```
**或者**：给 PlaymakerFocus 增加正面效果，例如组织核心的传球成功率额外 +5%，或关键传球（直塞）成功率提升。

#### A3. 增强 OffsideTrap 的收益
当前 OffsideTrap=0 的胜率反而高于 OffsideTrap=1/2，说明越位陷阱几乎没有收益。
```go
// 当前：offsideChance := 0.08 - float64(offsideTrap)*0.025
// 建议：offsideChance := 0.10 - float64(offsideTrap)*0.035
// 这样 offsideTrap=2 时，直塞球越位概率从 3% 提升到 3%，但 offsideTrap=0 时从 8% 提升到 10%
```

### 方案 B：削弱专精收益，增强均衡战术（中优先级）

当前均衡战术（T11）胜率偏低，说明"全点一项"的收益高于"平均分配"。

建议：
1. 引入**战术协同惩罚**：当多个参数同时达到极端值时，增加体力消耗或降低执行效率。例如高位逼抢(Pressing=4, DefLine=4)应该比单独 Pressing=4 更耗体力。
2. 增强**默认/均衡配置**的基础收益：例如所有参数都在 1-3 范围内时，球队获得 +5% 执行效率（控球矩阵微调 +0.02）。

### 方案 C：阵型平衡性调整（中优先级）

FormationBase 中某些阵型的区域权重差距过大：
- F04（Iron Wall）后场高但前场极低，导致整体控球很难推进。
- F07（Diamond Control）中场极强但前后场弱，容易被绕过中场直接打穿。

建议：
1. 缩小各阵型的极端区域权重差距，避免负值区域过于严重。
2. 引入**阵型克制循环**：例如 F04 后场强 → 克制 F03 前场强；F03 前场强 → 克制 F06 后场强。

### 方案 D：增加战术参数的可见性和反馈（低优先级，设计层面）

当前玩家无法直观看到战术参数对比赛的具体影响。建议：
1. 在比赛报告中增加"战术生效统计"：高位逼抢成功次数、越位陷阱触发次数、反击成功率等。
2. 在战术设置界面给出实时评估分数（基于当前球员属性与战术的匹配度）。

## 六、总结

| 维度 | 评价 | 优先级 |
|------|------|--------|
| 代码一致性 | 已修复 2 处不一致/反向 bug | ✅ 完成 |

1. 优先实施方案 A1-A3，修复极端选项的弱势问题。
2. 运行更大规模的回归测试（10,000+ 场）验证修复效果。
