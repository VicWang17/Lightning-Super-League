## 1. 引言

### 1.1 项目背景

随着足球运动的全球普及和电子竞技的兴起，足球经理类游戏受到了广大玩家的喜爱。本项目旨在打造一款与现实足球世界同步、玩法丰富、注重策略与社交的在线足球经理游戏——"闪电超级联赛"。玩家将扮演足球俱乐部经理，管理球队、参与比赛、进行球员交易，体验执掌豪门的乐趣。

### 1.2 项目目标

*   打造一款高度拟真、数据驱动的在线八人制足球经理游戏。
*   提供公平、平衡的游戏环境，鼓励策略竞技。
*   建立活跃的玩家社区，促进玩家间的互动与交流。
*   实现持续的内容更新与迭代，保持游戏活力。

### 1.3 目标用户

*   **核心用户**：深度足球迷，对足球战术、球员数据有深入了解的玩家。
*   **泛用户**：喜欢模拟经营、策略竞技类游戏的玩家。
*   **年龄段**：18-45岁，男性用户为主。

### 1.4 设计理念

**"真实的足球世界，有趣的游戏体验"**

本游戏追求**拟真与游戏性的黄金平衡**。这是一款**八人制足球经理游戏**，场上8人（1门将+7 outfield）、名单上限15人。经济系统参照现实足球俱乐部的收支结构，通过"工资帽+财政健康度"机制确保数值可控；青训系统提供两条并行的年轻球员获取通道——**自主青训**（投入预算，观察成长，赛季末签约）和**选秀大会**（联赛内志愿优先级自动分配）；交易系统采用"公开拍卖+自由市场"的双层流转，确保市场流动性。

核心设计原则：
1. **策略优先**：长期规划优于短期投机，理性经营优于烧钱堆砌。
2. **底层友好**：新手和低级别联赛玩家拥有明确的上升通道，专注青训可实现"小球队逆袭"。
3. **市场驱动**：球员身价由市场供需决定，系统仅提供估值参考和底价保护。
4. **流转通畅**：青训未签约球员→选秀→拍卖→自由市场，形成完整的球员流通链路。

## 2. 设计风格

### 2.1 视觉风格

*   **现代、简洁、专业**：界面设计以黑色、亮蓝色和深绿色为主，营造年轻化、专业的足球氛围。
*   **扁平化设计**：图标、按钮等元素采用扁平化风格，提升界面的现代感和易用性。
*   **数据可视化**：球员能力、球队数据、比赛统计等信息采用图表、雷达图等可视化方式呈现，直观易懂。
*   **高质量图片素材**：球员头像、队徽、球场等采用高质量图片，提升游戏整体质感。

### 2.2 交互风格

*   **直观易上手**：核心操作流程清晰明了，新手引导完善，帮助玩家快速熟悉游戏。
*   **高效便捷**：常用功能入口明显，减少操作层级，提升操作效率。
*   **信息反馈及时**：玩家的每一个操作都能得到及时的系统反馈，增强操作的确认感。
*   **沉浸式体验**：比赛日的文字直播、关键事件的动画效果等，营造紧张刺激的比赛氛围。

#### 3.2.2 年龄系统

**核心规则**：数据库中存储**相对出生偏移量**（负数），球员年龄随赛季序号自然增长。

**计算公式**：

```
当前年龄 = 当前赛季序号(0起始) + |出生偏移量|

示例：
- 出生偏移量 = -20
- 第1个赛季(序号0)：年龄 = 0 + 20 = 20岁
- 第5个赛季(序号4)：年龄 = 4 + 20 = 24岁
- 第10个赛季(序号9)：年龄 = 9 + 20 = 29岁
```

**年龄与游戏机制的关联**：

| 年龄段 | 标签 | 年龄系数(工资) | 成长特点 | 关键节点 |
|--------|------|---------------|---------|---------|
| ≤20岁 | 新秀 | 0.6 | 快速成长 | 青训营低工资 |
| 21-25岁 | 黄金期 | 1.0 | 稳定成长 | 最佳竞技状态 |
| 26-28岁 | 成熟期 | 1.1 | 成长放缓 | 巅峰能力值 |
| 29-30岁 | 下滑期 | 0.9 | 不再成长，开始缓慢下降 | 能力值每年-1 |
| 31-33岁 | 老将 | 0.7 | 能力下降加速 | 每年-2 |
| ≥34岁 | 生涯末期 | 0.5 | 急剧下降 | 每年-3，考虑退役 |

**退役机制**：
- 球员年龄 ≥ 36岁时，每赛季末有50%概率自动退役。
- 年龄 ≥ 38岁时，强制退役。
- 退役后球员从游戏中移除，但保留历史数据记录。

#### 3.2.4 综合能力（OVERALL）

综合能力（OVR）根据球员**位置**采用不同的权重公式计算，确保每个位置的评分都有意义。

**计算公式**：

```
OVR = Σ((各项属性 / 20) × 该位置权重)

即：每项属性先除以20转换为百分比（0.05-1.0），再乘以该位置的权重系数，求和后得到0-100的综合评分。四舍五入取整。
```

**示例**：一名前锋的射门18、头球16、力量15、速度17、盘带14、爆发力16、传球8，其余属性一般：
- OVR_FW = (18/20)×20 + (16/20)×10 + (15/20)×10 + (17/20)×18 + (14/20)×15 + (16/20)×10 + (8/20)×3 + (10/20)×3 + (10/20)×3 + (10/20)×5 + (8/20)×3
- = 18 + 8 + 7.5 + 15.3 + 10.5 + 8 + 1.2 + 1.5 + 1.5 + 2.5 + 1.2 = **75分（优秀级）**

*   **引擎联动**：OVR不直接参与比赛事件判定（事件判定使用具体属性），但影响球员被选中参与事件的概率权重。高OVR球员更容易被选中参与关键事件，体现"球星效应"。

**各位置权重表**：

| 属性 | FW(前锋) | MF(中场) | DF(后卫) | GK(门将) |
|------|---------|---------|---------|---------|
| 射门 | 20% | 2% | 0% | 0% |
| 传球 | 3% | 18% | 5% | 5% |
| 盘带 | 15% | 12% | 0% | 0% |
| 速度 | 18% | 7% | 12% | 0% |
| 力量 | 10% | 2% | 18% | 0% |
| 体能 | 3% | 15% | 12% | 0% |
| 防守 | 0% | 10% | 24% | 0% |
| 头球 | 10% | 0% | 12% | 0% |
| 视野 | 0% | 14% | 0% | 0% |
| 抢断 | 0% | 7% | 8% | 0% |
| 爆发力 | 10% | 2% | 0% | 0% |
| 传中 | 3% | 8% | 5% | 0% |
| 控球 | 0% | 8% | 0% | 0% |
| 远射 | 5% | 5% | 0% | 0% |
| 平衡 | 3% | 0% | 4% | 0% |
| 出击 | 0% | 0% | 0% | 15% |
| 扑救 | 0% | 0% | 0% | 40% |
| 反应 | 0% | 0% | 0% | 30% |
| 站位 | 0% | 0% | 0% | 15% |

> **说明**：门将的OVR由扑救(40%)、反应(30%)、站位(15%)、出击(10%)、传球(5%)五门将属性决定。计算公式与 outfield 球员一致：各项属性先除以20转换为百分比，再乘以权重求和。

**OVR分档标签**：

| OVR范围 | 标签 | 定位 |

| 1-19 | 业余 | 极低水平 |

#### 3.2.5 球员特性（招牌技能系统 v5.0）

> **设计目标**：让每名球员拥有独特的场上标签，增加比赛的叙事性和不可预测性。招牌技能在比赛中特定条件下概率触发，提供属性加成或特殊效果，使比赛内容更有层次。

**3.2.5.1 机制概述**

- 每名球员可拥有 **1-2个** 招牌技能（青训/转会市场生成时随机分配）。
- 技能分三个稀有度：**普通（Common）/ 稀有（Rare）/ 传奇（Legendary）**。
- 稀有度决定触发频率和效果强度：普通(效果+3%~+5%)、稀有(+5%~+8%)、传奇(+8%~+12%，可改变事件走向)。
- 技能不会随成长改变，生来固有（少数可通过特殊道具或事件重置）。
- 负面技能（玻璃体质）随机出现在约15%的球员中。

#### 3.2.6 性格系统（完全隐藏）

> **核心设计**：球员的六种性格对工资-表现关系有截然不同的影响，但**性格本身对玩家完全不可见**。玩家只能通过签约时的一句模糊描述和长期观察球员表现来猜测性格。这是游戏最核心的"识人"乐趣——你永远不会100%确定一个新签约的球员到底是什么性格。

**3.2.6.1 性格类型与工资-表现关系**

六种性格决定了球员在不同工资水平下的表现波动幅度：

| 性格 | 签约时的一句话暗示 | 高工资(≥120%) | 标准(90-119%) | 低工资(<90%) | 波动幅度 |
|------|-------------------|--------------|--------------|-------------|---------|
| **拜金型** | "他好像很在意钱。" | **+6%** 🔥 | +2% | **-6%** ❌ | 极大(±6%) |
| **野心型** | "他好像很在意钱。" | **+5%** 🔥 | +2% | **-5%** ❌ | 很大(±5%) |
| **职业型** | 无暗示 | +4% | +2% | -3% ⚠️ | 中等(±3%) |
| **激情型** | "他很热爱足球。" | +4% | +2% | -2% | 中等(±2%) |
| **忠诚型** | "他很热爱足球。" | +3% | +1% | -1% | 很小(±1%) |
| **团队型** | "他很热爱足球。" | +3% | +1% | -1% | 很小(±1%) |

> **关键设计**：拜金型球员给高工资时**最拼命(+6%)**，给低工资时**最摆烂(-6%)**——工资对他的影响是决定性的。团队型给高工资也只+3%，给低工资只-1%——他对工资不敏感。玩家需要根据这个差异来判断性格。

**3.2.6.2 性格对玩家完全不可见**

| 信息 | 可见度 | 说明 |
|------|--------|------|
| **性格类型** | **完全不可见** | 球员档案中不显示任何性格标签 |
| **具体数值** | **完全不可见** | 胃口系数等数值从不展示 |
| **签约暗示** | **仅签约时可见** | 只有一句话："他好像很在意钱" 或 "他很热爱足球" |
| **表现反馈** | **始终可见** | 球员状态（火热/低迷）是唯一能观察性格的线索 |

> **猜测方法**：签约一个新球员后，先给标准工资(100%)观察3场比赛——如果状态一直是🔥火热，大概率是忠诚/团队型；如果状态是➖平淡或↓低迷，可能是拜金/野心型。然后调整工资验证猜测：加薪到120%，如果状态飙升到🔥，确认是拜金型；如果状态没变，可能是团队型。

**3.2.6.3 性格与续约**

- **忠诚型/团队型**：续约时工资效果额外+1%（给100%也+2%）。
- **拜金型/野心型**：续约时无额外加成，必须给更高工资才能维持状态。
- **暗示显示**：续约时同样显示那句话暗示，帮助玩家判断是否值得留。

#### 3.2.7 球员状态

> **可见度设计**：球员状态对玩家**只显示视觉标签**（颜色和箭头），不显示具体百分比数值。这避免了玩家陷入"精确计算"，保留了直觉判断的空间。

**3.2.7.1 状态视觉标签**

| 状态 | 视觉显示 | 效果（玩家不可见的后台数值） |
|------|---------|---------------------------|
| **火热** | 🔥 红色火焰 | +5% |
| **良好** | ↑ 绿色向上箭头 | +2% |
| **平淡** | ➖ 灰色横线 | 0% |
| **低迷** | ↓ 黄色向下箭头 | -5% |
| **受伤** | 🚑 医疗图标 | 不能出场 |
| **停赛** | 🚫 禁止图标 | 不能出场 |
| **疲劳** | 😓 疲劳图标 | -5% |

| 满意度 | 视觉显示 | 效果（后台数值） |
|--------|---------|----------------|
| ≥120% | 🔥 火焰 + "拼命" | +5% |
| 100-119% | ✅ 绿色 + "满意" | +2% |
| 85-99% | ➖ 灰色 + "平常心" | 0% |
| 70-84% | ⚠️ 黄色 + "不满" | -3% |
| <70% | ❌ 红色 + "愤怒" | -5% |

**3.2.7.2 状态来源与观察线索**

球员状态由两个因素共同决定，但**只有比赛表现状态直接可见**：

1. **比赛表现状态**（直接可见）：基于最近2-3场比赛评分，显示🔥/↑/➖/↓。
2. **工资驱动加成**（后台隐藏）：由签约时的工资比例+性格决定，**不单独显示**，但会影响实际比赛表现。

> **观察性格的线索**：如果你发现一个球员明明OVR不低，但连续多场比赛状态低迷(↓)，尝试给他加薪——如果加薪后下一场立刻变火热(🔥)，那他很可能是拜金型。如果加薪后状态变化很小，那他可能忠诚/团队型。

**3.2.7.3 状态持续性**

| 状态类型 | 持续时间 | 说明 |
|---------|---------|------|
| **比赛表现** | 每场重新计算 | 基于最近2-3场比赛评分 |
| **受伤** | 随机天数 | 轻伤1-3场，重伤5-10场 |
| **停赛** | 固定场次 | 黄牌累计1场，红牌2-3场 |
连续3场首发后触发 |

#### 3.2.8 球员成长与衰退

**3.2.8.1 潜力评级可见度**

每名球员有一个隐藏的**潜力上限值**（Potential, 1-100），决定其最终能成长到多强。玩家**不能直接看到这个数值**，只能通过以下线索推断：

| 可见信息 | 显示方式 | 说明 |
|---------|---------|------|
| **潜力字母** | **始终可见** | 以字母显示：S(80-100)、A(60-79)、B(40-59)、C(20-39)、D(1-19) |
| **成长速度** | **青训球员可见** | 快/中/慢，间接反映潜力高低 |
| **潜力上限值** | **始终隐藏** | 具体数值不显示，仅系统内部使用 |
| **OVR** | **始终可见** | 综合能力精确到整数 |

> **设计意图**：潜力字母给玩家一个粗略预期（"这个球员潜力是A级"），但不给精确数值，保留"开盲盒"的探索乐趣。青训球员的成长速度（快=高潜力，慢=低潜力）提供额外判断依据。

**各字母对应的潜力范围与预期**：

| 字母 | 潜力范围 | 预期成年OVR | 稀有度 |
|------|---------|------------|--------|
| **S** | 80-100 | 75-100 | 极稀有（约2%） |
| **A** | 60-79 | 60-80 | 稀有（约8%） |
| **B** | 40-59 | 45-65 | 较少（约25%） |
| **C** | 20-39 | 30-50 | 常见（约35%） |
| **D** | 1-19 | 20-35 | 常见（约30%） |

**3.2.8.2 成长阶段**

| 年龄段 | 成长类型 | 说明 |
|--------|---------|------|
| ≤20岁 | 快速成长期 | 训练效果+30%，比赛经验+50% |
| 21-25岁 | 稳定成长期 | 训练效果+15%，比赛经验+30% |
| 26-28岁 | 成熟期 | 训练效果+5%，比赛经验+10% |
| 29-30岁 | 停滞期 | 训练效果0%，不再自然成长 |
| 31-33岁 | 衰退期 | 每年最弱的2项属性-1 |
| ≥34岁 | 加速衰退期 | 每年最弱的3项属性-2 |

**3.2.8.3 训练成长**

- 日常训练可选择训练重点，对应属性获得小幅提升（每次+0.5~1.5点）。
- 专项训练可针对个别球员的弱项进行补强。
- 成长上限受**潜力上限**约束，达到上限后不再成长。

**3.2.8.4 比赛经验成长**

- 每场比赛后，出场球员获得经验值。
- 表现优秀的球员（评分≥7.5）获得额外经验加成。
- 经验积累到一定程度转化为属性点提升。

#### 3.2.9 球员系统的数据流转

球员数据在系统中流经以下路径：

```
数据库球员表
    ↓
比赛引擎 —— 读取属性+特性+状态 → 模拟比赛事件
    ↓
交易系统 —— 读取属性+年龄 → 计算系统估值
    ↓
经济系统 —— 读取OVR+年龄+联赛 → 计算工资
    ↓
青训系统 —— 生成时分配隐藏潜力 → 成长后更新属性
    ↓
新闻系统 —— 读取特性+性格 → 生成个性化新闻
```

| **逼抢强度** | 5档 | 从不逼抢 ←→ 全场疯抢 | 逼抢事件权重↑，抢断成功率+3%，体能消耗+15%，身后空当+10% |
| **防守收缩** | 3档 | 保持阵型 ←→ 极度收缩 | 收缩→中路防守密度↑，区域控制度+5%，边路空当↑ |
| **盯人策略** | 3档 | 区域防守 ←→ 人盯人 | 人盯人→指定对方核心接球成功率-8%，阵型可能被拉扯 |
| **越位陷阱** | 3档 | 从不 ←→ 积极造越位 | 积极→越位陷阱成功率↑，身后空当风险+15% |
| **犯规侵略性** | 4档 | 文明 ←→ 凶悍 | 凶悍→抢断成功率+5%，犯规概率+20%，黄牌风险+15% |

*   **球员个人指令**：每个位置可设置2-3条个人指令，让核心球星拥有独特战术角色。

| 位置 | 可选指令 | 效果 |
|------|----------|------|
| **FW** | 积极前插 / 回撤接应 / 拉边游走 / 内切射门 | 前插→禁区事件权重+10%；回撤→中场接球概率↑；内切→禁区射门权重↑ |
| **MF** | 拖后组织 / 前压逼抢 / 边路支援 / 远射 | 拖后→后场传导权重↑；逼抢→中场抢断权重↑；远射→禁区外射门权重+8% |
| **DF** | 上抢压迫 / 深度回撤 / 出球组织 / 边路助攻 | 上抢→前场抢断权重↑；出球→后场传导精度+5%；助攻→边路传中权重↑ |
| **GK** | 出击型 / 门线型 / 参与出球 | 出击→单刀防守+10%；出球→后场传导+3% |

*   **临场战术调整**：比赛中可通过折叠面板实时调整6项战术，让经理人根据场上局势随时改变球队打法。

| 可调项目 | 档位 | 说明 |
|----------|:--:|:---------|
| **比赛心态** | 5档 | 保守控球 ←→ 全力进攻，影响进攻事件整体权重±10% |
| **进攻宽度** | 5档 | 收缩中路 ←→ 拉开边路，实时改变区域控制度分布 |
| **防线高度** | 5档 | 低位防线 ←→ 高位防线，影响直塞/过顶球被反击风险 |
| **逼抢强度** | 5档 | 被动防守 ←→ 全场疯抢，实时改变逼抢事件权重与体能消耗 |
| **传中策略** | 5档 | 低平球 ←→ 高球传中，影响传中事件的分支概率 |
| **换人** | — | 从替补席选择球员替换场上球员（与常规换人共用3次额度） |

**调整规则**：
- 每次战术调整后进入 **5分钟冷却**，冷却期内不可再次调整任何战术项。
- **中场休息** 可无条件调整一次，不计冷却，且允许临时变更阵型。
- 整场比赛（不含中场休息）最多调整 **6次**。
- 换人最多 **3次**，与赛前设定及伤退强制换人共用额度。

**快速预设**：系统提供三个一键方案，覆盖最常见的临场应变场景：
- **加强进攻**：进攻宽度+2档、防线高度+2档、逼抢强度+1档、传中策略高球+1档
- **稳住防守**：防线高度-2档、逼抢强度-1档、进攻宽度-1档、比赛心态偏保守
- **全力一搏**：比赛心态最大、进攻宽度最大、逼抢全场疯抢、防线前提至最高（建议最后15分钟且落后时使用）

**调整过渡**：战术指令变更不是瞬间生效，而是通过后续 **5-10个事件** 平滑过渡。引擎在过渡期内逐步将新战术参数混入事件概率计算，避免"变脸式"的不真实感；过渡期间解说词会穿插"调整阵型""改变打法"等叙事提示。

*   **战术影响**：战术设置通过**区域控制度模型**直接影响比赛引擎：
  1. **阵型**决定基础区域控制度分布（30%权重）
  2. **球员属性**通过热区映射修正控制度（45%权重）
  3. **战术指令**提供动态修正（20%权重）
  4. **比赛动态**（进球/红牌/体能）实时调整（10%权重）

  最终控制度矩阵输入引擎，决定事件概率分布、攻守走向和比赛节奏。

#### 3.3.3 训练系统

> **设计目标**：将训练系统打造为玩家每日必访的核心玩法入口。通过真实的"时段制训练+一周规划"机制，让玩家感受到"每天调度球队"的管理乐趣；通过丰富的训练类型和快速方案降低上手门槛，同时保留深度策略空间。

**3.3.3.3 疲劳系统**

疲劳是训练系统的**核心约束资源**。每名球员有独立的疲劳值（0-100）：

| 疲劳区间 | 状态 | 影响 |
|---------|------|------|
| 0-30 | 精力充沛 | 训练效果+10%，比赛状态+2% |
| 31-60 | 正常 | 无影响 |
| 61-80 | 疲劳 | 训练效果-20%，受伤风险+10% |
| 81-95 | 极度疲劳 | 训练效果-40%，受伤风险+25%，比赛属性-3% |
| 96-100 | 透支 | **强制休息**，无法安排任何训练，比赛属性-5% |

**疲劳变化规则**：
- 每场比赛后：所有出场球员疲劳+20-40（视出场时间和比赛强度）。
- 每天未安排训练/休息：疲劳自然恢复-5。
- "全队休息"：疲劳-25。
- "轻度拉伸"：疲劳-10。
- "按摩恢复"：指定3人疲劳-30。
- 高强度训练（射门/体能/对抗）：疲劳+15-25。
- 中等强度训练（战术/传球）：疲劳+8-12。
- 低强度训练（录像/课堂）：疲劳+3-5。

**受伤风险**：
- 疲劳>80时进行高强度训练，有5%概率触发球员受伤（伤停1-3场）。
- 连续3天高强度无恢复，全队受伤风险+10%。

**3.3.3.4 训练效果机制**

**短期效果**（影响下场比赛）：
- 战术训练的效果在比赛当天生效，持续1场。
- "进攻战术演练"→下场比赛全队射术+1%。
- "防守站位训练"→下场比赛全队防守+1%。
- 同类战术训练不叠加，取最后一次。

**长期效果**（属性成长）：
- 技术训练在每次完成后，对应属性微量增长（+0.1~0.3）。
- 成长上限受潜力约束。
- 21岁以下球员技术训练效果+50%（成长黄金期）。

**效果衰减**：
- 同一训练连续3天重复，第4天起效果-50%。
- "传球特训"连续做4天 → 第4天只+0.15（原+0.3）。
- 解决方式：轮换训练内容或插入休息。

**3.3.3.7 每日随机默认训练**

**防玩家不上线机制**：

每天0:00，系统自动为**当天未设置的时段**生成默认训练内容。

| 默认逻辑 | 规则 |
|---------|------|
| **生成时机** | 每天0:00检查，仅填充"空格子" |
| **不会覆盖** | 玩家已手动设置的格子不受影响 |
| **生成逻辑** | 基于当前球队状况智能选择： |
| - 疲劳高 | 优先生成"全队休息"或"轻度拉伸" |
| - 疲劳低 | 优先生成"体能特训"或"战术训练" |
| - 有比赛日 | 前一天自动生成针对性战术 |
| - 无比赛日 | 偏向技术训练或恢复 |
| - 连续3天同类 | 自动轮换不同类型 |

**默认训练效果**：
- 默认生成的训练效果为手动设置的**80%**。
- 例如：手动设置"射门特训"效果+0.3，默认生成+0.24。
- 这 incentivizes 玩家每天上线调整，但不会让不上线的玩家"亏太多"。

**上线提醒**：
- 如果今天有自动生成的默认训练，玩家上线时弹窗提示："今日有X项训练由系统自动安排，效果为手动的80%。建议调整以获得最佳效果。"

**3.3.3.8 训练系统与比赛引擎的联动（v5.0）**

- **体能储备→初始体能**：训练后累积的疲劳值会降低球员下场比赛的初始体能。初始体能公式：`初始体能 = 85 + (STA属性-10)×1.5 - 疲劳值×0.3`。疲劳值越高，比赛开局体能越低，影响全场表现（详见3.5.1引擎体能系统）。
- **赛前备战训练→战术执行度**：赛前"战术演练"训练提升球队战术执行度，为战术指令效果提供+3%的额外修正。
- **定位球训练→定位球事件加成**：赛前"定位球训练"在下场比赛中临时提升任意球/角球/点球相关事件的成功率+5%。
- **按摩恢复→体能恢复加成**：赛前"按摩"训练为球员提供+3%的中场休息恢复加成。
- **射门/防守特训→事件权重加成**：射门特训→下场比赛禁区射门事件权重+5%；防守特训→铲球/逼抢事件权重+3%。
- **体能特训→体能池上限**：体能特训可临时提升体能池上限+3点（仅影响下一场），让高体能球员在加时赛仍有竞争力。
- **点球特训→点球大战加成**：赛前进行点球特训的球队，在点球大战中每名球员点球命中率+3%。
- **赛前最后训练**：比赛日前一天自动推荐的训练，会根据对手特点调整。
  - 对手进攻强 → 推荐"防守站位训练"（提升防守事件权重+3%）。
  - 对手防守弱 → 推荐"进攻战术演练"（提升进攻事件权重+3%）。
- **赛后疲劳结算**：比赛结束后全队疲劳大幅上升，系统推荐次日安排恢复训练。
- **连胜/连败调整**：3连胜后系统推荐"战术课堂"防止松懈；3连败后推荐"心理放松"+"录像分析"。

- **训练疲劳与比赛体能衰减循环**：
  ```
  训练 → 累积疲劳 → 初始体能降低 → 比赛中更早进入疲劳区间 → 属性衰减 → 表现下降 → 需要更多恢复训练
  ```
  这个循环是训练系统的核心策略深度——玩家必须在"提升能力"和"保持体能"之间找到平衡。

### 3.4 交易系统

#### 3.4.0 核心设计哲学

交易系统采用**"公开拍卖市场 + 自由市场"**的双层流转体系：
- **公开拍卖市场**：长期开放，是球员流转的主要渠道，价格由竞价决定。
- **自由市场**：拍卖到期无人出价的球员进入自由市场，以固定价格随时签约。

**防作弊设计原则**：
1. **底价保护**：任何球员不能以低于系统估值一定比例的价格成交。
2. **市场透明**：所有交易记录公开可查。
3. **青训球员保护**：青训出品球员出售有底价减免保护，但提拔后有一定冷却期。

#### 3.4.1 系统估值体系

每笔交易的前提是系统对球员的"公允估值"。该估值**仅作为参考底价**，不限制市场成交价的上限。

**估值算法**：

```
系统估值 = 基础值 × 年龄系数 × 潜力系数 × 状态系数 × 联赛声望系数 × 稀有度系数
```

| 因子 | 说明 | 计算公式 |
|------|------|----------|
| **基础值** | 由球员OVR决定 | OVR×10万（基础单位）。例如OVR 80的球员，基础值 = 800万 |
| **年龄系数** | 年龄越大系数越低 | ≤20岁: 1.20; 21-23岁: 1.10; 24-26岁: 1.00; 27-28岁: 0.85; 29-30岁: 0.65; 31-32岁: 0.45; ≥33岁: 0.30 |
| **潜力系数** | 由潜力评级决定 | 潜力S(80-100): 1.50; A(60-79): 1.25; B(40-59): 1.10; C(20-39): 1.00; D(1-19): 0.85 |
| **状态系数** | 当前状态 | 火热: 1.15; 良好: 1.00; 平淡: 0.95; 低迷: 0.85; 受伤停赛中: 0.75 |
| **联赛声望系数** | 所在联赛级别 | 顶级联赛: 1.30; 次级联赛: 1.15; 三级联赛: 1.00; 四级联赛: 0.85 |
| **稀有度系数** | 市场供需自动调节 | 供不应求时上浮1.05-1.20; 供过于求时下浮0.80-0.95 |

**估值更新频率**：每赛季自动重算2次（赛季中、赛季末），球员重大表现后即时微调。

#### 3.4.2 公开拍卖市场

**3.4.2.1 挂牌流程（卖家）**

1. **选择球员**：从阵容中选择要出售的球员（必须不在伤停名单中，且入队满48小时）。
2. **设定参数**：
   - **起拍价**：不得低于系统估值的 `MAX(50%, 20万)`。底价保护线为估值的50%，但最低不低于20万。
   - **一口价（可选）**：不得低于起拍价的1.5倍。
   - **拍卖时长**：可选择24小时、48小时（默认）、72小时。
3. **上架费用**：挂牌时收取起拍价的 `2%` 作为上架费（最低1万，最高50万），不退还。
4. **成交**：拍卖结束时，若有≥1个有效出价，则最高价者得。无人出价则**流拍，进入自由市场**。
5. **成交手续费**：成交后收取成交价的 `5%` 作为交易税。

> **青训球员底价减免**：青训出品的球员，首次挂牌时起拍价可低至系统估值的 `35%`（而非普通球员的50%）。这是为了鼓励青训流通，让中小球队能以更低门槛出售青训成果。

**3.4.2.2 竞价流程（买家）**

1. **浏览与搜索**：支持按位置、能力范围、年龄范围、价格范围筛选。
2. **关注**：最多关注20人。
3. **出价规则**：
   - 首次出价 ≥ 起拍价。
   - 加价幅度：当前价的 `5%-20%`。
4. **资金冻结**：出价时冻结对应金额，被超越则自动解冻。
5. **代理出价**：玩家可设最高心理价位，系统自动以最小加价幅度代为竞价。
6. **倒计时保护**：最后10秒内有人出价，倒计时重置为10秒，最多重置5次。
7. **成交**：最高出价者赢得球员。

**3.4.2.3 市场调控**

| 调控类型 | 触发条件 | 调控措施 |
|----------|----------|----------|
| **价格熔断** | 成交价超过系统估值的300% | 限制该球员72小时内再次上架 |
| **低价保护** | 成交价低于系统估值的60% | 标记为异常交易，计入监控 |
| **上架冷却** | 同一球员7天内只能上架1次 | — |
| **市场注入** | 某位置供应严重不足（<5个挂牌） | NPC释放一批AI球员 |
| **市场吸收** | 某位置供应严重过剩（>30个挂牌） | NPC以估值的60%-80%批量收购 |

#### 3.4.3 自由市场

**3.4.3.1 系统定位**

自由市场是拍卖市场的**下游承接层**——所有未能在拍卖市场成交的球员，最终都会进入自由市场，以一口价形式供玩家随时签约。这确保没有任何球员会"烂在系统里"，市场始终保持流动性。

**3.4.3.2 进入自由市场的球员来源**

| 来源 | 说明 | 初始定价 |
|------|------|----------|
| **拍卖流拍** | 拍卖到期无人出价 | 系统估值的60% |
| **选秀未签约/落选** | 选秀大会中未被任何球队签约的球员 | 系统估值的50%（新秀折扣） |
| **解约球员** | 玩家主动解约的球员 | 系统估值的55% |
| **系统回收再投放** | 长期未成交的系统AI球员 | 系统估值的50%-70% |

**3.4.3.3 签约规则**

- **签约方式**：一口价直接购买，无需竞价。
- **签约环节**：一口价购买后，进入签约界面设定年限和工资（详见3.9签约系统）。
- **数量限制**：每个球队每48小时内最多从自由市场签约2名球员，防止批量抢购。
- **价格更新**：自由市场价格每24小时自动下调5%（最低降至系统估值的30%），直到被签约或系统回收。

#### 3.4.4 交易日志与公示

- 所有交易完成后生成记录，公示24小时。
- 记录包含：球员名、成交价/签约价、系统估值、买卖双方。
- 玩家可查看最近50笔全服交易。

#### 3.4.5 租借系统

- 玩家可将球员挂牌租借，设定期限（半个赛季或一个赛季）。
- 租借方承担球员全部工资，可选择是否支付租借费。
- 租借球员不能再次转租。

### 3.5 比赛系统（v5.0 事件驱动模拟引擎）

> **设计目标**：将比赛从简单的概率模拟升级为事件驱动模拟引擎，每场比赛由一连串具体的足球事件构成，玩家的战术设置和球员指令能够影响事件概率和走向。

#### 3.5.1 模拟引擎架构

**1. 比赛基本信息**

| 参数 | 设定 | 说明 |
|------|------|------|
| **制式** | 八人制 | 场上8人（1门将+7 outfield） |
| **比赛时长** | 50分钟 | 25分钟/半场，每半场约20-30个事件 |
| **中场休息** | 10分钟 | 体能恢复30%+STA加成 |
| **换人** | 最多3人 | 伤退/体能耗尽/战术调整时触发 |
| **加时赛** | 2×10分钟 | 淘汰赛需要分出胜负时触发 |
| **点球大战** | 5轮+突然死亡 | 加时后仍平局时触发 |

**2. 事件驱动核心机制**

引擎基于**事件链(Event Chain)**驱动比赛，而非传统的概率公式直接产出比分。

```
比赛流程：开球 → 事件A → 判定结果 → 事件B → 判定结果 → ... → 进球/失球/半场结束
```

每个事件包含四个要素：`(事件类型, 参与球员, 判定对抗, 时间)`。

**事件库总览**：引擎共定义 **59种具体事件类型**，覆盖从开球到终场哨的完整足球叙事。事件分为六大类：
- **球权转换事件**：开球、球门球、界外球、角球、任意球（直接/间接）、点球、争顶后球权转换
- **传导球事件**（核心主体，占60-70%）：后场传导、中场传导、安全短传、回传、横传调度、长传调度、门将短传/手抛球、三角传递、套边短传、一脚出球、控球调整
- **进攻推进事件**：边路突破、内切突破、直塞球、挑传/过顶球、二过一配合、交叉跑位、下底传中、45度斜传、远射、禁区射门、头球攻门、凌空抽射、带球推进、禁区混战射门
- **防守对抗事件**：逼抢拦截、铲球抢断、封堵射门、补位防守、头球解围、争顶对抗、包夹防守、协同逼抢、回追防守、拦截传球、门将出击、门将扑救
- **特殊事件**：犯规（普通/恶意）、黄牌、红牌、越位、伤退、换人
- **比赛流程事件**：上半场开始/结束、中场休息、下半场开始/结束、常规时间结束

**传球距离定义**：引擎中传球不按物理距离划分，而按区域关系定义：
- **短传/传导**：同区域内的传递，依赖PAS+VIS+CON
- **长传/转移**：跨区域传递，依赖PAS+STR+VIS

**传球距离定义**：引擎中传球不按物理距离划分，而按区域关系定义：
- **短传/传导**：同区域（如后场中央→后场中央、左中场→左中场）的传递，依赖PAS+VIS，体能消耗低（0.5~0.8），高频率出现
- **长传/转移**：跨区域（如后场→前场、左路→右路）的传递，依赖PAS+STR+VIS，体能消耗中高（1.2~1.5），改变攻防节奏

**多人协同事件**：由2-4名球员协同完成的复合事件，丰富比赛叙事层次：
- 进攻多人事件：二过一配合、交叉跑位、三角传递、快速反击协同推进
- 防守多人事件：包夹防守、协同逼抢、双人争顶、补位协防

**核心流程**（每5-30秒产生一个事件）：
1. **读取当前状态**：球权归属、球所在区域、双方控制度矩阵、球员体能状态
2. **生成候选事件**：根据区域控制度、战术权重、事件冷却，生成5-10个候选事件
3. **计算概率**：`P(事件) = 基础权重 × 控制度修正 × 战术修正 × 冷却修正`
4. **选择事件**：加权随机选择一个事件
5. **选取参与球员**：从当前区域及相邻区域，根据球员能力权重和体能状态选取攻防双方球员
6. **执行判定对抗**：使用18种数值对抗公式计算结果，sigmoid映射为成功/失败概率，保留概率性而非确定性
7. **更新状态并重新计算候选池**：更新球权、区域、控制度等状态，根据新的 `active_zone` 重新计算候选事件池（执行 Step 1-6），加权随机抽取下一个事件。极少数强制连带（如传中→争顶、射正→扑救、进球→开球）直接触发。
8. **累加统计**：更新控球率、射门、传球等统计数据
9. **推送文字**：生成文字直播语句

**3. 九区域系统**

球场被划分为3×3网格区域，每个区域有独立的控制度值（-1.0 ~ +1.0），正值表示进攻方控制，负值表示防守方控制。

```
        左                 中                 右
前场 (0,0) [左翼]    (0,1) [禁区]       (0,2) [右翼]
中场 (1,0) [左中场]   (1,1) [中圈]       (1,2) [右中场]
后场 (2,0) [左后场]   (2,1) [后场中央]    (2,2) [右后场]
```

**区域事件权重矩阵**：每个区域有不同的事件发生倾向。

| 区域 | 典型进攻事件 | 典型防守事件 |
|------|-------------|-------------|
| (0,1)禁区 | 禁区射门、头球争顶、补射 | 门将扑救、封堵射门、后卫争顶 |
| (1,1)中圈 | 中场传导、直塞球、二过一 | 逼抢拦截、预判拦截、补位防守 |
| (2,1)后场中央 | 后场传导、长传转移 | 逼抢拦截、头球解围 |
| (1,0)/(1,2)边路 | 边路突破、传中、内切突破 | 边路防守、铲球 |

**4. 区域控制度模型**

区域控制度是引擎的核心输入，由四层叠加计算：

```
Δ_effective[r,c] = Δ_formation[r,c] × 0.30  (阵型基础)
                   + Δ_players[r,c] × 0.45  (球员热区)
                   + Δ_tactics[r,c] × 0.20   (战术修正)
                   + Δ_dynamic[r,c] × 0.10   (动态修正)
```

*   **阵型控制度(30%)**：每种阵型有预设的区域控制度分布（详见3.3.2阵型表）
*   **球员控制度(45%)**：根据球员位置的Primary Zone和Secondary Zone分布计算
    *   球员热区映射：GK在(2,1)附近；CB覆盖(2,0)-(2,2)；WF在(0,0)/(0,2)；ST在(0,1)
*   **战术修正(20%)**：
    *   传球倾向→传导/转移事件权重修正
    *   进攻宽度→边路/中路控制度±5%
    *   逼抢强度→前场/中场控制度±5%
    *   防线高度→前后场控制度再分配
    *   核心策略→指定球员所在区域控制度+5%
    *   个人指令→对应区域±3%~±10%
*   **动态修正(10%)**：
    *   进球后领先方控制度+5%（心理优势）
    *   红牌后少打一人方所有区域控制度×(6/7)
    *   体能低于30%的球员不再贡献控制度
    *   最后10分钟落后方控制度+3%（搏命）

**5. 事件触发概率模型**

候选事件概率计算：

```
P(event) = BaseWeight[event] × ControlFactor × TacticModifier × CooldownFactor × StaminaFactor

ControlFactor = 1 + (attack_control - 0.5) × 0.4  (攻击方控制度越高，进攻事件概率越高)
TacticModifier = 战术指令对特定事件的加成/惩罚
CooldownFactor = 同一事件在N秒内再次发生的概率衰减
StaminaFactor = 参与球员平均体能修正（体能低→高难度事件概率↓）
```

**6. 数值对抗核心公式（概率型，18种）**

所有对抗使用**概率型判定框架**，不是简单的比大小。通过sigmoid函数将属性差值映射为成功概率：

```
O = Σ(进攻方属性_i × 权重_i) × 体能系数 × 战术加成 × 技能加成
D = Σ(防守方属性_j × 权重_j) × 体能系数 × 战术加成 × 技能加成
Δ = O - D
P_success = sigmoid(Δ / 5.0) = 1 / (1 + e^(-Δ/5))

关键概率参考：
  Δ = 0   → P = 50%（势均力敌）
  Δ = +5  → P ≈ 73%（进攻方明显优势）
  Δ = +10 → P ≈ 88%（进攻方很大优势）
  Δ = +15 → P ≈ 95%（近乎必胜，但仍有小概率失败）
  Δ = -5  → P ≈ 27%（进攻方劣势）
```

然后随机抽样 `r = uniform(0,1)` 判定结果：

| 结果 | 条件 | 说明 |
|:------:|:--------:|:----|
| 成功 | r < P_success | 进攻方达成预期效果 |
| 失败 | r ≥ P_success | 进攻方未达成预期效果 |

> **v3修正**：删除「多级结果分级」四级划分，仅保留「成功/失败」两级。射门事件的细分差异（进球/射正/射偏/被封堵）由事件自身的 outcomes 定义。事件成功的"程度"不由通用结果分级决定，而由后续事件链的走向体现（如传中成功→争顶→射门进球 vs 传中成功→争顶→解围）。

> **设计意图**：强队vs弱队时Δ通常为+3~+8，胜率约65%-73%，体现"强队赢球概率高但不保证"。即使是巨大优势（Δ=+15），仍有约5%的失败概率，保留爆冷的戏剧性。

**8. 特殊规则系统**

*   **越位判定**：直塞球/过顶球/长传转移后触发越位检查。防线高度（战术指令）影响越位概率：高位防线→越位概率降低，低位防线→越位概率升高。判定公式：`P_offside = sigmoid((def_line_height - forward_speed) / 5)`。

*   **伤退系统**：铲球后4%、恶意犯规后8%、争顶后2%基础受伤概率。轻伤70%（坚持比赛，属性-10%），重伤30%（强制换人）。
*   **少打一人影响**：红牌后所有区域控制度×(6/7)，体能消耗+15%，反击被断概率+20%。

### 3.7 青训系统（v4.0 简化版）

> **设计目标**：建立一条清晰的年轻球员培养通道。玩家通过提高预算投入，在赛季中获得系统刷新的青训球员，观察他们在青训营中的自动成长来判断潜力，赛季末决定签约哪些人进入一线队。未签约的青训球员进入选秀大会，形成完整的球员流转链路。

#### 3.7.0 核心设计哲学

**青训 = 预算投入 → 系统刷新新人 → 自动成长 → 观察判断潜力 → 赛季末签约 → 进入一线队**

这是一条**单线直通**的培养链路，没有球探搜索的模糊信息，没有青年联赛的额外操作。玩家只需要做两件事：(1) 赛季初决定投入多少预算；(2) 赛季末决定签哪些人。

**成长即信息**：球员的潜力不直接显示。玩家通过观察青训营中球员的成长速度来**推断**其潜力——成长越快的球员，潜力上限越高。这模拟了现实中球探通过观察年轻球员的表现来判断其未来的过程。

**球员流转链路**：

```
系统刷新 → 进入青训营 → 自动成长 → 赛季末
    ↓
玩家签约 → 进入新人合同签约界面
    ↓
玩家放弃 → 进入选秀大会（本联赛内分配）
    ↓
选秀未签约/落选 → 进入拍卖市场
    ↓
拍卖流拍 → 进入自由市场（随时一口价签约）
```

#### 3.7.1 青训投入与刷新机制

**3.7.1.1 投入比例**

每赛季开始前（在预算规划中），玩家设定**"青训投入比例"**，范围 `5%-25%`。这笔钱从本赛季总预算中划拨，进入**青训专项基金**。青训基金用于：
- 支付刷新球员的签约费（每入营1名球员消耗一定基金）
- 维持青训设施运转

**投入比例的效果**：

| 投入比例 | 每次刷新人数 | 高潜球员概率 | 赛季2次刷新合计 |
|----------|-------------|-------------|----------------|
| 5% | 2人 | 低（约15%出现3星+） | 4人 |
| 10% | 3人 | 中（约25%出现3星+） | 6人 |
| 15% | 4人 | 中高（约35%出现3星+） | 8人 |
| 20% | 5人 | 高（约45%出现3星+） | 10人 |
| 25% | 6人 | 很高（约55%出现3星+） | 12人 |

> 青训营同时最多容纳 `8人`。如果营内已满8人，新的刷新不会到来，直到有球员被签约或放弃腾出位置。

**3.7.1.2 刷新时间点**

| 赛季天数 | 事件 |
|----------|------|
| **第4天** | **第一次刷新**：系统根据投入比例向青训营推送2-6名新球员 |
| **第8天** | **第二次刷新**：再次推送2-6名新球员 |

每次刷新时，系统会：
1. 检查青训营是否有空位。
2. 根据投入比例决定推送人数。
3. 生成球员：年龄14-17岁，初始属性5-10（对应OVR约25-40），隐藏潜力随机分配。
4. 球员直接进入青训营，开始自动成长。
5. 玩家收到"青训营有新球员加入"的通知。

**3.7.1.3 刷新球员的初始信息**

新进入青训营的球员，玩家可以看到以下信息：
- 年龄、位置、惯用脚
- **当前能力值**（显示具体数值）
- **成长速度**（显示为"快/中/慢"三个档位）
- 无明显优势/劣势（可选，增加一点个性）

**不显示的信息**：
- 具体潜力上限值（隐藏）
- 星级（隐藏）

> 玩家需要通过观察成长速度来判断潜力。成长速度快的球员 = 潜力高。

#### 3.7.2 青训营自动成长机制

**3.7.2.1 成长公式**

青训营中的球员，**每2游戏天**（约每周）自动进行一次成长结算：

```
属性增长 = 基础增长 × 设施加成 × 年龄加成 × 随机因子

其中：
- 基础增长 = (隐藏潜力/100 - 当前OVR/100) × 档位系数
  档位系数：快(高潜力 60-100): 0.8 | 中(中潜力 40-59): 0.35 | 慢(低潜力 1-39): 0.12
  （越接近潜力上限，增长越慢）
- 设施加成 = 1.0（1级）~ 1.5（5级）
  设施等级由玩家投资建设，详见3.8.5
- 年龄加成 = U15: 1.3, U16: 1.2, U17: 1.1, U18: 1.0, U19+: 0.8
- 随机因子 = 0.7 ~ 1.3（正态分布，模拟个体差异）
```

**成长速度档位（属性1-20制）**：
- **快**：每次增长0.4-0.8点属性 → 对应高潜力（60-100）
- **中**：每次增长0.15-0.35点属性 → 对应中等潜力（40-59）
- **慢**：每次增长0.0-0.12点属性 → 对应低潜力（1-39）

> 一赛季约10次成长结算，快档球员单个属性可增长4-8点，OVR可提升15-30分。

> 成长速度在球员入营时就由隐藏潜力决定，之后不会改变。所以玩家观察2-3个周期（约4-6天）就能大致判断一个球员的潜力档位。

**3.7.2.2 球员在营时长**

- 球员在青训营中最多待到赛季末（第26天）。
- 赛季末时，所有在营球员必须被处理：签约进入一线队，或放弃进入选秀大会。
- 未满18岁的球员如果未被签约，会**继续留在青训营**（进入下赛季），不强制处理。
- 已满18岁的球员如果未被签约，则**强制放弃**，进入选秀大会。

**3.7.2.3 成长曲线可视化**

青训营页面中，每个球员显示一条**成长曲线图**：
- X轴：时间（赛季天数）
- Y轴：OVR
- 曲线显示球员入营以来的能力变化
- 玩家可以直观地看到谁在"起飞"（曲线陡峭），谁在"停滞"（曲线平缓）

#### 3.7.3 赛季末签约决策

**3.7.3.1 青训签约截止日**

**赛季第23天23:59**是青训营球员签约的最终截止时间。此时青训营中的所有球员（除U18以下可选择继续留营的），玩家必须做出决定：

| 操作 | 说明 | 结果 |
|------|------|------|
| **签约** | 将球员签入一线队 | 占用阵容名额，进入新人合同签约界面 |
| **放弃** | 不签约 | 该球员进入选秀大会（18岁+强制放弃；U18可选放弃或留营） |

> **关键时间点**：
> - 第22天：系统弹窗提醒"青训签约截止倒计时2天"。
> - 第23天23:59：截止。超时未处理的球员**自动视为放弃**，进入选秀大会。
> - 第24天：选秀大会开始。

**3.7.3.2 签约限制**

签约青训球员受以下限制：
1. **阵容上限**：一线队总人数不能超过15人（含青训签约球员，八人制标准）。
2. **工资帽**：签约后的球员工资总额不能超过工资帽。青训球员签的是**新人合同**（估值为正常的40%，浮动70%-130%，详见3.9签约系统）。
3. **签约上限**：每赛季最多从青训营签约 `4人`。这是为了防止青训产出过度挤占转会市场空间。

**3.7.3.3 新人合同**

青训球员签约时签订的是**"新人合同"**，有以下特点：

| 属性 | 新人合同 | 正常合同对比 |
|------|----------|-------------|
| **工资** | 固定工资表的 **40%** | 正常合同的100% |
| **合同年限** | 固定 **2年** | 正常合同1-4年可变 |
| **底价保护** | 挂牌出售时起拍价可低至估值的 **35%** | 正常为50% |
| **续约** | 合同到期前自动触发续约结算，续约工资按正常工资表计算 | — |
| **解约** | 无违约金，随时可解约 | — |

> **新人合同的意义**：青训球员进入一线队的前两年是"试用期"，工资极低，大幅减轻球队工资压力。这为底层球队用青训球员替换高薪老将提供了强烈的经济激励。

#### 3.7.4 选秀大会（联赛内志愿优先级分配）

**3.7.4.1 选秀球员来源**

选秀大会的球员来源：
1. **本联赛各队放弃的青训球员**（第23天截止后自动汇总）。
2. **选秀志愿阶段中未被选中的球员**（落选秀）。

```
选秀池 = 联赛内所有球队"放弃的青训球员" + 上季落选秀
```

选秀大会**按联赛分别举办**——每个联赛（32个联赛各自独立）在赛季末举行自己的选秀大会。选秀球员只在同联赛的球队之间分配。

**3.7.4.2 选秀流程**

**Step 1：志愿排序（第22天开放，第23天截止）**

- 第22天，选秀池预览开放，玩家可拖拽排序志愿。
- 第23天23:59，志愿排序截止。
- 第23天同时也是青训签约截止日——青训放弃的球员在截止后立即进入选秀池。

**Step 2：选秀大会（第24天）**

- 第24天0:00，系统根据志愿排序+蛇形轮次自动分配球员。
- 分配结果一次性公布，所有球队同时收到通知。

- 选秀池生成后（第22天），玩家可以在选秀页面看到本联赛所有进入选秀池的球员列表。
- 球员信息**完全公开**：年龄、位置、当前能力、成长速度、在青训营的表现数据。
- 玩家通过**拖拽排序**，将自己感兴趣的球员按优先级从高到低排列。
- 玩家可以选择**"不选"**（将某些球员标记为不感兴趣，系统不会为其分配）。
- 志愿排序在第24天选秀大会开始前截止。

**Step 2：自动分配（第24天）**

- 系统采用**"志愿优先级+蛇形轮次"**算法自动分配球员：

```
分配算法：
1. 所有参与选秀的球队，按本赛季联赛排名倒序排列（第8名先选，第1名后选）
2. 第1轮：从第8名开始，依次查看其第1志愿
   - 若该球员未被分配且该球队未达上限 → 分配给该球队
   - 若已被分配 → 查看第2志愿，以此类推
3. 第2轮：从第1名开始（蛇形回头），重复上述流程
4. 直到所有球员被分配或所有球队达到上限
```

- 每支球队每赛季选秀最多可获得 **2人**。
- 分配结果在选秀大会当天一次性公布，所有玩家同时收到通知。

**Step 3：签约决策（第24-25天）**

- 分配成功后，球员进入球队的"待签约"名单。
- 玩家有24小时（到第25天）决定：
  - **签约**：进入新人合同签约界面（估值40%，浮动70%-130%，合同2年）。
  - **拒绝**：放弃签约，球员进入**拍卖市场**（以系统估值的50%起拍价上架）。
- 超时未做决定的，系统自动**拒绝**，球员进入拍卖市场。

**3.7.4.3 选秀球员的交易保护**

- 选秀签约的球员，60天内不能上架拍卖市场。
- 60天后可正常交易，但不享受青训出品的底价减免。

#### 3.7.5 青训设施系统

虽然删除了球探系统和青年联赛，但**青训设施**仍然保留——它决定了青训营中球员的成长速度。

| 设施等级 | 升级费用 | 每赛季维护费 | 成长速度加成 | 说明 |
|----------|----------|-------------|-------------|------|
| 1级（简陋） | 初始拥有 | 5万/赛季 | 基准 | 初始状态 |
| 2级（标准） | 60万 | 10万/赛季 | +15% | 早期可负担 |
| 3级（专业） | 40万 | 8万/赛季 | +30% | 中期目标 |
| 4级（先进） | 100万 | 15万/赛季 | +50% | 高级投资 |
| 5级（顶级） | 250万 | 30万/赛季 | +75% | 长期目标 |

> 设施维护费从**青训基金**中自动扣除。如果青训基金不足以支付维护费，设施效果降级运行（效果打7折）。

---

#### 3.7.6 青训系统的乐趣设计

1. **"观察+判断"的策略感**：玩家不是直接看到星级，而是通过成长曲线来判断谁值得签。这需要观察和思考，比直接给星级更有参与感。
2. **"低成本试错"的友好性**：即使判断错误签了一个成长慢的球员，新人合同的工资极低，解约无违约金，试错成本很低。
3. **"赌对了"的巨大回报**：如果一个成长快的球员一路培养到高能力，其出售价值可能是投入的数十倍。
4. **选秀的保底+意外**：即使自己青训判断失误，选秀大会还有机会捡到别人放弃的好苗子。
5. **完整的流转感**：看到自己放弃的球员在选秀中被别人签走，或在拍卖市场被买走，有一种"送自己的孩子出门"的叙事感。

### 3.8 经济系统

#### 3.8.0 核心设计哲学

**经济系统的三大支柱**：

1. **收支双轨清晰化**：收入来源和支出项目都有明确的计算公式。
2. **工资帽约束**：球员工资总额不能超过收入的一定比例，超支受限制。
3. **预算分配抉择**：每赛季初，玩家在有限预算中分配"转会预算"、"青训投入"、"工资预留"和"应急储备"。

#### 3.8.1 收入体系

**3.8.1.1 收入总览**

| 收入类别 | 占比（顶级联赛） | 占比（四级联赛） | 说明 |
|----------|-----------------|-----------------|------|
| **转播收入** | 35%-40% | 20%-25% | 由联赛级别决定 |
| **商业赞助** | 20%-25% | 10%-15% | 可更换赞助商 |
| **比赛日收入** | 20%-25% | 30%-40% | 可调整票价 |
| **杯赛奖金** | 5%-8% | 8%-12% | 取决于杯赛成绩 |
| **联赛分红** | 10%-15% | 10%-15% | 取决于联赛排名 |

**3.8.1.2 转播收入**

```
转播收入 = 联赛基础转播费 × 球队声望系数

- 顶级联赛：35万/赛季
- 次级联赛：28万/赛季
- 三级联赛：22万/赛季
- 四级联赛：18万/赛季

球队声望系数：0.8 ~ 1.3
```

**3.8.1.3 赞助商系统**

> **设计目标**：赞助商系统要简单到玩家"几乎一定会签"，同时通过"签约赞助商"这个行为将真实Web广告自然地融入游戏，成为开发者的盈利来源。不同类型的赞助商提供不同的收入曲线，让玩家有策略选择，但差距不大，不影响"以下克上"的公平性。

**3.8.1.3.1 签约窗口**

| 阶段 | 时间 | 说明 |
|------|------|------|
| **赞助商选择开放** | 上赛季第20天 | 系统推送5个赞助商选项 |
| **选择截止** | 本赛季第1天 | 未选择的球队自动签约默认赞助商 |
| **合同生效** | 本赛季全程 | 收入按合同规则发放 |

> **默认行为**：系统默认勾选"自动签约推荐赞助商"。不手动操作的玩家会在第1天自动获得一个赞助商，不会错过收入。

**3.8.1.3.2 赞助商类型**

赞助商分为两种类型，玩家从5个随机选项中选择：

| 类型 | 机制 | 特点 | 适合 |
|------|------|------|------|
| **稳定型** | 赛季初一次性支付全额赞助费 | 收入确定，无风险 | 稳健经营、新赛季不确定的球队 |
| **激进型** | 支付底金 + 按胜场/进球追加奖金 | 上限略高，有风险 | 自信能赢很多比赛的强队 |

> **差距控制**：激进型最高期望收入仅比稳定型高10%-15%，不会因为选错赞助商而导致经济差距过大。

**3.8.1.3.3 赞助商选项生成**

每个赛季，系统为每支球队**随机生成5个赞助商选项**：

```
5个选项 = 2个稳定型 + 2个激进型 + 1个惊喜型（可能是高底金激进型或高稳定型）
```

选项参数由球队联赛级别和声望决定：
- 顶级联赛：稳定型8-12万，激进型底金4万+每场胜场1.5万
- 次级联赛：稳定型6-9万，激进型底金3万+每场胜场1.2万
- 三级联赛：稳定型4-6万，激进型底金2万+每场胜场0.8万
- 四级联赛：稳定型3-4万，激进型底金1.5万+每场胜场0.5万

**3.8.1.3.4 赞助商展示**

- **不显示真实品牌名**：选项以图标+代号展示（如 🔷 品牌A、🔶 品牌B、🟢 品牌C），避免版权问题和审美疲劳。
- **显示关键信息**：类型（稳定/激进）、预计收入范围、合同条款摘要。
- **推荐标签**：系统根据球队上赛季表现推荐一个选项（"基于您上赛季12胜的表现，激进型预计收益更高"）。

**3.8.1.3.5 签约与广告植入**

**签约行为**：
- 玩家点击"签约"按钮，选择1个赞助商，合同立即生效。
- 未手动选择的球队在第1天自动签约推荐选项。

**广告植入（开发者盈利）**：
- 签约赞助商的球队，其**球场页面**和**比赛页面**会显示真实Web广告位。
- 广告以"球场广告牌"的形式融入UI（如场地四周的虚拟广告牌、记分牌旁的品牌展示）。
- **玩家心理**：通过"签约赞助商换取收入"这个游戏内行为，玩家将广告视为"合理的交易"而非"打扰"。
- 未签约赞助商的球队：页面无广告，但收入减少。

> **设计意图**：将现实中的"球场广告"概念引入游戏，既增加了真实感，又为开发者提供了可持续的盈利方式。玩家因获得了赞助收入而对广告的存在有心理接受度。

**3.8.1.3.6 收入发放**

| 类型 | 发放时间 | 金额 |
|------|---------|------|
| **稳定型** | 赛季第1天一次性发放 | 全额 |
| **激进型底金** | 赛季第1天发放 | 底金部分 |
| **激进型奖金** | 每场比赛后即时结算 | 胜场奖金/进球奖金 |
| **赛季末结算** | 赛季末根据总胜场/总进球补发 | 追加奖金 |

**3.8.1.3.7 赞助商与游戏性平衡**

- **差距控制**：赞助商收入仅占总收入的15%-25%，不会决定胜负。
- **策略性**：强队可选激进型搏上限，弱队可选稳定型保下限——但差距很小。
- **公平性**：所有球队看到的5个选项期望值相近，不会出现"别人拿到好赞助而我只能拿差的"。

|----------|---------|-----------|

| 标准 | ×1.0 | 基准 |
| 高价 | ×1.4 | -15% |

#### 3.8.2 支出体系

**3.8.2.1 支出总览**

| 支出类别 | 占比 | 说明 |
|----------|------|------|
| **球员工资** | 50%-65% | 最大支出，受工资帽限制 |
| **青训投入** | 5%-25%（可配置） | 战略支出 |
| **设施维护** | 3%-8% | 球场维护 + 青训设施 |
| **转会支出** | 不固定 | 一次性支出 |
| **其他** | 2%-3% | 差旅、医疗等 |

**3.8.2.2 工资帽机制**

```
球员工资帽 = 赛季总收入 × 工资帽比例

默认工资帽比例 = 65%（B级财务基准）
```

| 状态 | 触发条件 | 影响 |
|------|----------|------|
| **健康** | 实际工资 ≤ 工资帽 | 正常运营 |
| **轻度超支** | 工资帽 < 实际工资 ≤ 1.15倍 | 赞助商-10%，不能签新赞助商 |
| **严重超支** | 实际工资 > 1.15倍 | 禁止拍卖出价，青训投入强制5% |
| **破产边缘** | 连续2赛季工资 > 1.3倍 | 自动出售最高薪球员 |

**3.8.2.3 设施维护费**

- 球场维护：每年5-20万（与球场等级挂钩）。
- 青训设施维护：包含在青训基金中（从基金自动扣除）。

**3.8.2.4 转会支出**

- 一次性成本，直接从现金余额扣除。
- 不计入工资帽，但过度支出会导致现金枯竭。

#### 3.8.3 预算规划系统

**3.8.3.1 设置时间窗口**

| 时间节点 | 事件 |
|----------|------|
| **上赛季第20天** | 预算规划界面开放 |
| **上赛季第25天** | 预算规划截止，未设置者沿用上赛季比例 |
| **本赛季第1天** | 预算锁定，新赛季开始 |

**3.8.3.2 预算分配**

```
本赛季总预算 = 上赛季结余现金 + 本赛季预计收入

分配：
1. 转会预算：____%（建议20%-40%）
2. 青训投入比例：____%（5%-25%）
3. 工资预留：____%（建议55%-70%）
4. 应急储备：____%（建议5%-10%）
```

总和必须 = 100%。系统提供"建议分配方案"一键应用。

**3.8.3.3 应急储备**

- 建议预留5%-10%。
- 用途：意外支出（球员加薪要求、追加转会等）。
- 未使用部分结转至下赛季。

#### 3.8.4 财务健康度评级

| 评级 | 条件 | 影响 |
|------|------|------|
| **A（优秀）** | 连续盈利≥2赛季 + 工资≤0.8倍帽 + 储备≥20%收入 | 赞助+20%，球员工资帽+5% |
| **B（健康）** | 盈利或微亏 + 工资≤帽 | 正常运营 |
| **C（紧张）** | 亏损 + 工资>0.95倍帽 | 赞助-15%，不能签高级赞助商 |
| **D（危险）** | 严重亏损 + 工资>1.1倍帽 | 强制卖人，禁止拍卖出价，青训强制5% |

#### 3.8.5 财政公平原则（FFP）

```
滚动三年转会净投入 = 近三年买入总和 - 近三年卖出总和

若 滚动三年转会净投入 > 近三年总收入的30%：
  → 下赛季转会预算缩减50%
  → 连续2赛季触发则球员工资帽缩减10%
```

**豁免**：出售青训球员收入不计入FFP计算。

### 3.9 签约与工资系统

> **设计目标**：建立"系统估值 + 玩家浮动 + 性格反馈"的三层签约机制。工资不只是成本，更是激励球员的手段——给得合适，球员拼命；给得吝啬，球员摆烂。性格系统让不同球员对工资的反应截然不同，增加了识人善用的策略深度。

---

#### 3.9.1 签约触发时机

**球员进入签约界面的触发场景**：

| 场景 | 触发时机 | 默认处理 |
|------|---------|---------|
| **拍卖市场成交** | 倒计时结束、最高出价者赢得球员时 | 24小时内未操作，自动按估值100%签约，年限2年 |
| **询价成交** | 任一方接受报价时 | 24小时内未操作，自动按估值100%签约，年限2年 |
| **自由市场签约** | 玩家点击"一口价购买"时 | 立即进入签约界面 |
| **青训提拔** | 赛季末玩家点击"签约"时 | 进入新人合同签约界面 |
| **选秀签约** | 选秀大会分配结果公布后 | 24小时内未操作，自动按新人估值100%签约 |

> **默认机制**：所有签约都有**24小时窗口**。玩家在线时进入签约界面操作；离线时系统等待24小时，超时自动按估值100%、年限2年签约。这保证球员不会无限期悬而未决，也不会因玩家偶尔不上线而卡住。

---

#### 3.9.2 签约流程

当球员进入签约界面时，玩家需要决定两个维度：

| 维度 | 范围 | 说明 |
|------|------|------|
| **签约年限** | 1-3年 | 年限越长球员越稳定，但未来工资风险越大 |
| **工资比例** | 估值的70%-130% | 玩家自由浮动，直接影响球员满意度 |

**签约界面操作步骤**：

1. **查看球员档案**：系统展示球员OVR、位置、年龄。
2. **查看估值**：系统给出"建议工资"（估值 = 基础工资表[OVR] × 联赛系数 × 年龄系数）。
3. **调整年限**：拖动选择器：1年 / 2年 / 3年。
4. **调整工资比例**：拖动滑块：70%-130%，以1%为单位。
5. **实时预览**（关键）：系统实时显示：
   - 实际工资金额（估值 × 比例）。
   - 签约后工资帽使用率。
   - **球员反应预测**（基于性格和工资比例）：
     - 😤 愤怒（红）/ 😐 平淡（黄）/ 🙂 满意（绿）/ 🤩 狂喜（蓝）。
   - **预期状态效果**：属性加成/减成预览。
6. **确认签约**：玩家确认后合同生效。

> **设计意图**：签约界面是玩家与球员建立关系的第一个触点。通过实时反馈，玩家可以直观地看到"多花10%工资能换来什么"。给核心球员130%工资让他拼命，给团队型球员80%工资他也不闹——这就是策略。

---

#### 3.9.3 工资估值模型

系统估值是玩家浮动的基准，完全由球员客观条件决定：

```
估值 = 基础工资表[OVR] × 联赛系数 × 年龄系数
```

**3.9.3.1 基础工资表**

| OVR | 赛季工资（参考） | OVR | 赛季工资（参考） |
|-----|----------------|-----|----------------|
| 30 | 1.5万 | 65 | 4.5万 |
| 40 | 2.5万 | 70 | 6万 |
| 50 | 3万 | 80 | 12万 |
| 55 | 3.5万 | 85 | 18万 |
| 60 | 4万 | 90 | 28万 |

**3.9.3.2 联赛系数**

| 联赛级别 | 系数 |
|----------|------|
| 顶级联赛 | 1.3 |
| 次级联赛 | 1.1 |
| 三级联赛 | 1.0 |
| 四级联赛 | 0.8 |

**3.9.3.3 年龄系数**

| 年龄段 | 系数 |
|--------|------|
| ≤20岁 | 0.6 |
| 21-25岁 | 1.0 |
| 26-28岁 | 1.1 |
| 29-30岁 | 0.9 |
| 31-33岁 | 0.7 |
| ≥34岁 | 0.5 |

**3.9.3.4 估值示例**

| 球员 | OVR | 年龄 | 联赛 | 估值 |
|------|-----|------|------|------|
| A | 80 | 24 | 顶级 | 12万×1.3×1.0 = **15.6万/赛季** |
| B | 80 | 24 | 四级 | 12万×0.8×1.0 = **9.6万/赛季** |
| C | 90 | 19 | 顶级 | 28万×1.3×0.6 = **21.8万/赛季** |

#### 3.9.4 工资比例与表现效果

工资比例对球员表现的影响完全由性格决定，性格本身对玩家不可见，只有签约时的一句话暗示。

**性格-工资效果对照表**（玩家不可见的后台机制）：

| 性格 | 签约暗示 | 高工资≥120% | 标准90-119% | 低工资<90% | 波动幅度 |
|------|---------|------------|------------|------------|---------|
| **拜金型** | "他好像很在意钱。" | **+6%** 🔥 | +2% | **-6%** ❌ | 极大(±6%) |
| **野心型** | "他好像很在意钱。" | **+5%** 🔥 | +2% | **-5%** ❌ | 很大(±5%) |
| **职业型** | 无暗示 | +4% | +2% | -3% ⚠️ | 中等(±3%) |
| **激情型** | "他很热爱足球。" | +4% | +2% | -2% | 中等(±2%) |
| **忠诚型** | "他很热爱足球。" | +3% | +1% | -1% | 很小(±1%) |
| **团队型** | "他很热爱足球。" | +3% | +1% | -1% | 很小(±1%) |

> **关键设计**：拜金型给高工资时最拼命(+6%)，给低工资时最摆烂(-6%)——工资是他的命根子。团队型给高工资只+3%，给低工资只-1%——他对工资不敏感。玩家需要通过观察来判断性格。

**如何猜测性格**：

1. **签约时**：留意那句话暗示——"在意钱"大概率是拜金/野心，"热爱足球"大概率是忠诚/团队/激情。
2. **给标准工资(100%)观察3场**：
   - 状态一直火热 → 大概率忠诚/团队型（他们给标准工资就+1%~+2%）
   - 状态平淡或低迷 → 可能是拜金/野心型（他们给标准工资只+2%或0%，如果比赛表现稍差就变成低迷）
3. **调整工资验证**：
   - 加薪到120%，如果下一场立刻火热 → 很可能是拜金型（+6%效果惊人）
   - 加薪到120%，如果状态变化不大 → 可能是团队型（+3%效果温和）
   - 降薪到80%，如果立刻摆烂 → 确认拜金型
   - 降薪到80%，如果状态变化不大 → 确认团队型

> **游戏乐趣**：性格是黑箱，玩家像现实中的教练一样——"这个前锋我给了他100%工资，他居然还摆烂...难道他是个拜金型？下次给他120%试试。

#### 3.9.5 新人合同

青训出品和选秀球员的特殊签约规则：

| 项目 | 规则 |
|------|------|
| **估值折扣** | 估值 = 正常估值 × 0.4（40%） |
| **浮动范围** | 估值的70%-130%，即正常工资的 **28%-52%** |
| **年限** | 固定2年，不可调整 |
| **解约** | 无违约金，随时可解约 |
| **续约** | 到期后按正常估值续约（大幅涨薪） |

**示例**：OVR 65、18岁、顶级联赛青训球员
- 正常估值 = 4.5万 × 1.3 × 0.6 = 3.5万
- 新人估值 = 3.5万 × 0.4 = 1.4万
- 给100% = 1.4万/赛季（团队型→拼命+5%）
- 正常外购同OVR = 6万/赛季（团队型给100%也拼命）
- **节省4.3倍工资获得同样效果**

> **新人合同是底层翻身的核武器**：低工资+高激励，让青训球员成为性价比最高的战斗力来源。

#### 3.9.6 续约系统（玩家主动）

> **核心设计**：续约不是自动的，由玩家主动决定。玩家可以选择提前续约，也可以等合同到期后重新签约，或放走球员。

**3.9.6.1 续约触发条件**

| 时机 | 操作 |
|------|------|
| **合同剩1年时** | 玩家可在球员档案页点击"发起续约"按钮，进入续约界面 |
| **合同剩14天时** | 系统弹窗提醒"球员合同即将到期" |
| **合同到期时** | 若未续约，球员变为自由身，进入自由市场 |

**3.9.6.2 续约年限规则**

续约的年限是**累加**的，不是覆盖：

| 当前剩余年限 | 续约选择 | 续约后总年限 | 说明 |
|-------------|---------|-------------|------|
| 剩1年 | 续约2年 | **总共再打2个赛季** | 当前赛季+下赛季 |
| 剩1年 | 续约3年 | **总共再打3个赛季** | 当前赛季+下赛季+下下赛季 |
| 剩2年 | 续约2年 | 当前2年不变 + 延长2年 | 实际为"在当前合同基础上追加" |

> **关键说明**：当球员合同还剩1年时发起"续约2年"，意味着球员在当前赛季结束后，还会再效力**2个完整赛季**（下赛季和下下赛季）。不是"1+2=3年"，而是"当前合同被新合同替换，新合同期限=续约年限"。

**3.9.6.3 续约流程**

续约使用与**签约完全相同的机制**：

1. **重新估值**：按球员当前OVR、年龄、联赛重新计算估值（能力提升→估值上涨）。
2. **忠诚度加成**：原球队续约时，工资效果额外+1%（给100%也+2%）。这 incentivizes 留住球员。
3. **玩家设定年限和浮动**：同签约流程（1-3年，70%-130%）。
4. **工资帽检查**：如果续约后总工资超过工资帽，续约**无法提交**（按钮灰色，提示超帽）。
5. **确认续约**：玩家确认后新合同生效。

**3.9.6.4 自动续约设置**

玩家可在设置中勾选**"合同剩14天时自动续约"**：

| 设置 | 行为 |
|------|------|
| **开启** | 合同剩14天时，系统自动按估值100%、年限2年发起续约。玩家上线后可查看/修改。 |
| **关闭（默认）** | 仅弹窗提醒，不自动续约，由玩家手动决定。 |

> **自动续约的边界**：即使开启自动续约，如果续约会导致超工资帽，系统自动跳过该球员，并发邮件提醒玩家手动处理。

**3.9.6.5 续约失败的邮件提醒**

以下情况续约无法完成：

| 失败原因 | 系统行为 |
|---------|---------|
| 超工资帽 | 续约界面提交时直接提示"工资帽不足"，无法提交 |
| 自动续约超帽 | 跳过自动续约，发送邮件："[球员名]续约因工资帽不足失败，请上线手动调整" |
| 合同到期仍未续约 | 发送邮件："[球员名]已成为自由身，进入自由市场" |

**邮件内容示例**：
> 📧 **续约失败提醒**
> 球员：[球员名]（OVR 82，前锋）
> 失败原因：续约后工资帽使用率将达112%
> 建议操作：出售高薪球员腾出空间，或选择不给该球员续约
> 截止日期：[日期]，逾期球员将变为自由身

---

#### 3.9.7 重新签约（合同到期后再签）

球员合同到期变为自由身后，球队可以选择：

| 操作 | 说明 |
|------|------|
| **重新签约** | 在自由市场找到该球员，以正常估值重新签约（年限1-3年，浮动70%-130%） |
| **放弃** | 不签约，球员留在自由市场供其他球队签约 |
| **抢签** | 其他球队可在自由市场抢先签约 |

> **重新签约 vs 续约的区别**：续约有忠诚度折扣（胃口-0.15），重新签约没有折扣，且可能面临竞价。

---

#### 3.9.8 工资帽机制

```
球员工资帽 = 赛季总收入 × 工资帽比例

工资帽比例：
- A级财务：70%
- B级财务：65%
- C级财务：60%
- D级财务：55%
```

| 状态 | 触发条件 | 影响 |
|------|----------|------|
| **健康** | 实际工资 ≤ 工资帽 | 正常运营 |
| **轻度超支** | 工资帽 < 实际工资 ≤ 1.1倍 | 赞助商-10%，不能签新赞助商 |
| **严重超支** | 实际工资 > 1.1倍 | 禁止拍卖出价，青训投入强制5% |
| **破产边缘** | 连续2赛季工资 > 1.2倍 | 自动出售最高薪球员 |

---

#### 3.9.9 工资帽实时监控

- 球队管理页面顶部显示**工资帽进度条**：
  - 🟢 绿色：≤70%
  - 🟡 黄色：70%-90%
  - 🔴 红色：>90%
- 每次签约/续约时，签约界面实时显示"签约后工资帽使用率将升至X%"。
- 工资构成饼图：核心球员/主力/轮换/替补/新人合同各占多少。

---

#### 3.9.10 签约系统的策略深度

**策略1：精打细算型**
> 识别团队型球员 → 给80%工资 → 拼命(+5%) → 工资帽压力小 → roster深度好

**策略2：明星驱动型**
> 核心球员给130% → 拼命(+5%) → 关键时刻carry → 但工资帽紧张 → 其他位置必须压缩

**策略3：中庸稳健型**
> 所有人给100% → 职业型满意(+2%)，其他平常心(0%) → 稳定但缺乏惊喜

**策略4：赌性格型**
> 新球员先给100%试探 → 观察3场后的状态反馈 → 判断是否调整 → 是拜金型就加到120%，是团队型保持80%

**策略5：压榨剥削型（高风险）**
> 所有人给80%最低 → 如果是拜金型则-6%摆烂 → 球队整体战力打折 → 恶性循环

### 3.10 联赛系统

*   **功能描述**：构建完整的联赛竞技体系，为玩家提供分级竞争环境，通过升降级机制实现球队实力的动态平衡和上升通道。
*   **联赛体系结构**：
    *   游戏共设有**四个联赛体系**（东区、西区、南区、北区），每个联赛体系相互独立，拥有完整的升降级链条。
    *   **单个联赛体系结构**（共64支球队，8个联赛）：
        *   **顶级联赛（1级）**：1个联赛，8支球队，最高水平的竞技舞台。
        *   **次级联赛（2级）**：1个联赛，8支球队，通往顶级联赛的桥梁。
        *   **三级联赛（3级）**：2个平行联赛（3A和3B），每个联赛8支球队。
        *   **四级联赛（4级）**：4个平行联赛（4A、4B、4C、4D），每个联赛8支球队，新手和弱势玩家的起始舞台。
    *   **全服总计**：4个联赛体系 × 8个联赛 = 32个联赛，256支球队。
*   **升降级规则**（每个联赛体系内部）：
    *   **顶级联赛 ↔ 次级联赛**：
        *   顶级联赛第1名（冠军）直升下赛季闪电杯小组赛种子。
        *   顶级联赛最后1名降级至次级联赛。
        *   顶级联赛第7名与次级联赛第2名进行附加赛，决定升级/降级名额。
        *   次级联赛第1名（冠军）直升顶级联赛。
    *   **次级联赛 ↔ 三级联赛**：
        *   次级联赛最后2名降级至三级联赛（根据规则分配至3A或3B）。
        *   三级联赛A和三级联赛B的冠军各直升1个名额，共2个名额升级至次级联赛。
        *   三级联赛A亚军 vs 三级联赛B亚军进行附加赛，胜者与次级联赛第7名进行附加赛，决定第3个升级名额。
    *   **三级联赛 ↔ 四级联赛**：
        *   三级联赛A ↔ 四级联赛A、四级联赛B（对应关系）。
        *   三级联赛B ↔ 四级联赛C、四级联赛D（对应关系）。
        *   四级联赛冠军直升对应三级联赛。
        *   两个对应四级联赛的亚军进行附加赛，胜者与对应三级联赛第7名进行附加赛，决定升降级名额。
*   **积分与排名规则**：
    *   胜场得3分，平局得1分，负场得0分。
    *   排名依据：积分 → 净胜球 → 进球数 → 相互对战成绩。
*   **联赛页面展示**：
    *   积分榜：实时显示各球队排名、场次、胜平负、进球/失球、净胜球、积分。
    *   赛程表：显示本赛季所有比赛的对阵和时间安排。
    *   射手榜/助攻榜：显示联赛个人数据排名。
    *   升降级区标识：直观显示降级危险区和升级竞争区。

### 3.11 杯赛系统

*   **功能描述**：提供杯赛玩法，为不同级别的球队提供竞技和争夺荣誉的机会。
*   **闪电杯（全服顶级杯赛）**：
    *   **参赛资格**：四个联赛体系的顶级联赛球队（4个联赛 × 8支球队 = 32支球队）。
    *   **赛制**：
        *   **小组赛**：8个小组，每组4队，单循环3轮（A-H组）。
        *   **淘汰赛**：小组前2名晋级16强，单淘汰制决出冠军。
        *   **总计**：3轮小组赛 + 4轮淘汰赛 = 7轮。
    *   **对阵规则**：16强交叉对阵，A组第1 vs B组第2，以此类推。
    *   **奖励**：丰厚奖金、特殊奖杯、联赛声望加成。
*   **杰尼杯（体系内杯赛）**：
    *   **参赛资格**：每个体系内除顶级联赛外的所有球队（次级8队 + 三级16队 + 四级32队 = 56支球队/体系）。
    *   **赛制**：
        *   **预选赛**：48支非种子球队（三级+四级），决出24支。
        *   **32强**：24支预选赛胜者 + 8支次级联赛种子球队（轮空）。
        *   **淘汰赛**：32强开始单淘汰制，共5轮。
        *   **总计**：1轮预选赛 + 5轮正赛 = 6轮。
    *   **奖励**：适中奖金、杰尼杯奖杯。
*   **杯赛页面展示**：对阵图、赛程表、历史记录。

### 3.12 赛季系统

*   **功能描述**：管理游戏的时间运转和赛季周期。
*   **赛季周期**：
    *   **单赛季时长**：26天为一个完整赛季。
    *   **休赛期**：赛季结束后2天休赛期，用于转会窗口、升降级结算、新赛季准备。
*   **比赛频率**：
    *   基本每天一场比赛。
    *   联赛单赛季14场（双循环，7个对手×2）。
    *   全服32个联赛共1,792场联赛比赛。

*   **定时调度系统**：
    *   赛季开始前自动生成完整赛程表。
    *   到达比赛时间点时，系统自动执行比赛模拟（玩家不在线也可进行）。
    *   所有玩家共享统一游戏时间线。
*   **赛季节点管理**：
    *   赛季开始：初始化数据、生成赛程、开启转会窗口、预算锁定。
    *   转会窗口：全赛季开放拍卖市场，休赛期开放自由市场。
    *   杯赛轮次：与联赛交错进行。
    *   赛季结束：结算排名、执行升降级、财务健康度评估、发放奖励。

*   **联盟/协会系统**：玩家组建或加入联盟。
*   **移动端优化与适配**。

## 7. 风险与挑战

*   **数值平衡性**：经济系统、青训系统、交易系统的数值需要反复测试和调优。建议alpha测试邀请50-100名玩家进行2-3个赛季的压力测试。
*   **服务器压力**：拍卖市场倒计时竞价需要高并发支持。
*   **外挂与作弊**：
    *   小号刷利：通过底价保护、同IP限制等设计层面措施降低风险。
    *   自动出价脚本：通过行为模式检测识别。
    *   数据篡改：所有关键计算在服务端完成。
*   **内容更新频率**：需要持续推出新的球员数据、活动和玩法。
*   **版权问题**：球员姓名、球队队徽等可能涉及版权，需妥善处理。
